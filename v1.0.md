# SNAB Specification v1.0

## Terminology
Before we get into the nitty-gritty, here are some terms that will be used in this specification.

- **Datastream**  
  All bytes following the SNAB header. The datastream contains the encoded structs and arrays.
- **Header**  
  The fixed-length, 16-byte prefix of a SNAB file that describes how the datastream should be interpreted.
- **Endianness**  
  The byte order used to encode multi-byte numeric values in the datastream.
- **Struct**  
  A collection of key-value pairs. Keys are UTF-8 null-terminated strings and must be unique within a struct.
- **Array**  
  An ordered collection of values without keys. Arrays may contain values of mixed datatypes.
- **Terminator**  
  A special datatype used to mark the end of a struct or array.
- **Null-Terminated**  
  Terminated by a zero-value byte (`0x00`), or for UTF-16 strings, by two zero bytes.

## Header
The SNAB header describes how the data inside should be interpreted. The header is always 16 bytes in length. The header is always encoded in little-endian.

| **Field**          | **Byte Size** | **Datatype** | **Example** | **Usage**                                            |
|--------------------|----------|--------------|-------------|------------------------------------------------------|
| Major Version      | `0x01`   | `ubyte`      | `1`         | Indicates the major version of the SNAB spec in use. |
| Minor Version      | `0x01`   | `ubyte`      | `0`         | Indicates the minor version of the SNAB spec in use. |
| Flags              | `0x02`   | `ushort`     | `0`         | Describes properties about the datastream. |
| Language Indicator | `0x04`   | `char[4]`    | `PY\00\00`  | Describes which language feature-set is included. |
| Checksum           | `0x04`   | `uint`       | ---         | A CRC32 checksum of the datastream. |
| Size               | `0x04`   | `uint`       | ---         | The size of the datastream before decompression (if applied). |

### Versioning
The major and minor version bytes will indicate the version of the reader that is required to parse the data of the current datastream.

### Flags
The flags describe how the data should be interpreted. Bit shifts are relative to the least-significant bit. All unused flags must be set to `0`.

| **Field**            | **Shift** | **Size** | **Usage**|
|----------------------|-----------|----------|----------------------------------------------------------------------------------------------------------|
| Endianness            | `0`       | `1`      | Determines the endianness of the datastream, where `0` is little-endian and `1` is big-endian. |
| Extended Feature Set | `1`       | `1`      | Determines whether the extended feature set of the language specifier is used in the file. |
| Compressed           | `2`       | `1`      | Determines whether the datastream is compressed or not. |


### Checksum
The checksum must be [zlib crc-32](https://github.com/luvit/zlib/blob/master/crc32.c) to ensure compatibility between all implementations. The checksum is based on the datastream after compression is applied and does not include the header.

## Core Datatypes
Below is a list of core datatypes, their indicator and their usage. The size of the datatype does not include the datatype or the value itself.
| **Datatype** | **Indicator** | **Size** | **Usage**|
|--------------|---------------|----------|------------------------------------------------------------|
| `terminator` | `0x00`        | ---      | Terminates the current struct or array that is being read. |
| `struct`     | `0x01`        | ---      | Identifies a struct. |
| `array`      | `0x02`        | ---      | Identifies an array. |
| `string`     | `0x03`        | ---      | A UTF-8 null-terminated string. |
| `w-string`   | `0x04`        | ---      | A UTF-16 null-terminated string, terminated by two null bytes. |
| `real`       | `0x05`        | `0x08`   | An IEEE 754 double precision float. |
| `int64`      | `0x06`        | `0x08`   | A 64-bit signed integer. |
| `false`      | `0x07`        | `0x00`   | False. |
| `true`       | `0x08`        | `0x00`   | True. |
| `undefined`  | `0x09`        | `0x00`   | An undefined value. |
| `null`       | `0x0A`        | `0x00`   | A null value. |
| `buffer`     | `0x0B`        | ---      | A binary buffer prefixed with a `uint32` length field. |
| `reserved`   | `0x0C` - `0x7F` | ---     | Reserved datatypes for future versions of SNAB. |

## Extended Datatypes
Each implementation can include extended datatypes for the language the implementation is written in and for user-definable datatypes. `0x80` - `0xFF` are reserved for extended datatypes.

## Datastream Structure
The root of the data must either be a struct or array with no key assigned to it. A terminator datatype may only be used to terminate a struct or array and must not appear as a member datatype. Structs contain key-value pairs and are structured as `[datatype][key][value]` where the key is a null-terminated UTF-8 string, and the value is determined by the datatype. Arrays are similar but do not include a key. The datatype is always 1 byte. Struct keys must be unique.

### Compression
The compression must use [zlib deflate](https://github.com/madler/zlib), of which many implementations exist for almost all major languages. The compressed datastream can use any level of compression that zlib deflate supports.
